## 2. 项目背景与目标
### 2.1 核心问题
当前「高效 Prompt 模板」模块为静态内容展示（写死在页面中），用户只能查看，无法沉淀、迭代与管理模板，导致常用 Prompt 无法复用与演进，团队/项目专用模板也难以快速新增与维护。

### 2.2 需求目标
1. 支持在「AI 提效指南」中维护可编辑的 Prompt 模板库（新增/编辑/删除/查看/复制）。
2. 提升复用效率：列表态只展示名称+描述，详情态才展示全文，降低信息噪音。
3. 适配当前项目数据持久化方式：以本地存储为主，刷新页面后数据仍可用。

## 3. 用户画像与场景（可选）
### 3.1 目标用户
- 个人使用者：沉淀个人常用 Prompt，持续迭代措辞与结构。
- 管理者/产品/研发：沉淀团队内固定格式模板（PRD/周报/复盘/代码评审等）。

### 3.2 用户场景
- 场景一：临时需要“需求澄清 Prompt”，在列表中找到目标模板并一键复制使用。
- 场景二：使用过程中发现更优表达，进入编辑并保存，后续可重复使用。
- 场景三（可选）：模板数量增多后，希望通过搜索快速定位（作为增强项规划）。

## 4. 功能需求详述
### 4.1 Prompt 模板列表（外层）
#### 4.1.1 用户故事/功能点 1.1：列表展示
- 作为：用户
- 我希望：在「AI 提效指南」右侧栏卡片中看到模板列表
- 以便于：快速浏览有哪些模板并进行下一步操作
- 业务规则与逻辑：
  1. 列表仅展示：名称（title）+ 描述（description），不在列表态展示内容全文（content）。
  2. 每条模板提供操作入口：查看/编辑、复制内容、删除（带二次确认）。
  3. 列表默认按更新时间倒序展示（updatedAt 新的在前）。
  4. 展示适配右侧栏窄列：标题单行展示，描述最多两行展示；超出部分截断省略。
  5. 列表容器支持滚动（避免模板数量增加导致页面过长），并提供“管理全部”入口进入更大视图进行编辑与维护。
- 验收标准：
  1. 进入「AI 提效指南」后可看到模板列表区域，且每项仅展示名称与描述。
  2. 每项均能看到“查看/编辑”“复制”“删除”入口。
  3. 模板数量较多时列表区域可滚动且不影响页面整体布局。
  4. 点击“管理全部”可进入大视图继续查看/新增/编辑/删除。

#### 4.1.2 用户故事/功能点 1.2：新增入口
- 作为：用户
- 我希望：点击“新增模板”进入编辑界面
- 以便于：快速创建一条新的 Prompt 模板
- 业务规则与逻辑：
  1. 列表顶部提供“新增模板”按钮。
  2. 新增进入的编辑界面字段为空（或提供可选默认值）。
- 验收标准：
  1. 点击“新增模板”能打开编辑界面。
  2. 未保存前，列表数据不发生变化。

### 4.2 模板编辑（新增/编辑）
#### 4.2.1 用户故事/功能点 2.1：新增模板
- 作为：用户
- 我希望：填写名称、描述与内容并保存
- 以便于：将常用 Prompt 固化为可复用的模板
- 业务规则与逻辑：
  1. 字段约束：
     - 名称：必填，<= 50 字；去除首尾空格；不可全为空白字符。
     - 描述：可选，<= 120 字；可为空。
     - 内容：必填，多行；不可全为空白字符。
  2. 保存成功后：
     - 自动生成唯一 id（例如时间戳+随机串）。
     - 写入 createdAt/updatedAt（ISO8601）。
     - 返回列表并刷新展示。
  3. 保存失败（校验不通过）应提示具体原因并阻止保存。
- 验收标准：
  1. 名称/内容为空时无法保存，并给出提示。
  2. 保存后新模板出现在列表中，刷新页面仍存在。

#### 4.2.2 用户故事/功能点 2.2：编辑模板
- 作为：用户
- 我希望：修改现有模板的名称、描述或内容并保存
- 以便于：持续迭代模板质量
- 业务规则与逻辑：
  1. 编辑入口来自列表项“查看/编辑”。
  2. 进入编辑界面时回填当前模板字段。
  3. 保存时更新 updatedAt，id 与 createdAt 不变。
  4. 编辑取消不应影响原数据。
- 验收标准：
  1. 修改并保存后，列表展示的名称/描述发生对应变化。
  2. 刷新页面后仍保持更新后的内容。

### 4.3 模板查看与复制
#### 4.3.1 用户故事/功能点 3.1：查看内容全文
- 作为：用户
- 我希望：在详情/编辑态看到模板内容全文
- 以便于：确认内容并决定是否复制或修改
- 业务规则与逻辑：
  1. 列表态不展示 content。
  2. 进入详情/编辑态后展示 content 全文（等宽字体、多行保留换行）。
- 验收标准：
  1. 在详情/编辑态能完整看到 content 内容，并保留换行。

#### 4.3.2 用户故事/功能点 3.2：一键复制内容
- 作为：用户
- 我希望：点击“复制内容”将模板内容复制到剪贴板
- 以便于：快速粘贴到 Chat/文档/工单中使用
- 业务规则与逻辑：
  1. 优先使用浏览器 Clipboard API 复制。
  2. 复制失败时给出错误提示并不导致页面异常。
  3. content 为空时复制按钮不可用或提示不可复制。
- 验收标准：
  1. 点击复制后，剪贴板内容与模板 content 一致。
  2. 复制失败时能看到明确提示（例如浏览器权限限制）。

### 4.4 模板删除
#### 4.4.1 用户故事/功能点 4.1：删除模板（二次确认）
- 作为：用户
- 我希望：删除不再需要的模板
- 以便于：保持模板库整洁
- 业务规则与逻辑：
  1. 删除前弹出二次确认提示，用户确认后才执行删除。
  2. 删除后不可恢复（MVP）。
  3. 删除成功后立刻从列表移除并写入存储。
- 验收标准：
  1. 未确认时不删除；确认后模板从列表消失。
  2. 刷新页面后被删除模板不会再出现。

### 4.5 数据模型与本地持久化
#### 4.5.1 用户故事/功能点 5.1：本地存储读写
- 作为：用户
- 我希望：模板数据在刷新页面后依旧存在
- 以便于：长期沉淀个人/团队模板
- 业务规则与逻辑：
  1. localStorage Key：`nexus_prompt_templates`。
  2. 存储内容：模板对象数组（JSON stringify）。
  3. 初始化策略：
     - 若本地为空：写入 2-3 条默认模板（从当前静态模板迁移而来）。
     - 若本地已有：直接加载，不覆盖用户数据。
- 验收标准：
  1. 新增/编辑/删除后刷新页面，状态与数据保持一致。

## 5. 边界条件与约束
1. 不做登录/权限体系（MVP）。
2. 不做多人协作、在线同步或云端存储（后续可扩展）。
3. 不做复杂标签体系与版本管理（后续可扩展）。
4. 字段约束：名称<=50字、描述<=120字；内容为多行文本；均需做空白校验与长度校验。

## 6. 非功能性需求（如适用）
- 性能需求：列表渲染与编辑操作在常见模板规模（例如 <= 500 条）下保持流畅，无明显卡顿。
- 安全性需求：不引入鉴权；提供 UI 提示“不要在模板中存放敏感信息（公司机密、用户隐私等）”；复制与存储不输出任何敏感日志。
- 易用性需求：新增/编辑/删除/复制均在 3 步以内完成；校验错误提示明确到字段级。
- 兼容性需求：使用标准 Web API；Clipboard API 不可用时给出兼容提示与降级策略（如提示手动复制）。

## 7. 数据指标（可选）
1. 模板库规模：模板条数、平均内容长度（仅定义口径；MVP 不强制埋点采集）。
2. 使用强度：复制次数、编辑次数（仅定义口径；后续如引入统计再实现）。
3. 成功指标：用户能在 1 分钟内完成“新增一条模板并复制使用”的闭环。

## 8. 待讨论/待明确事项
1. 编辑界面形态：MVP 默认采用 Drawer 抽屉（更适合长文本编辑），并与右侧栏“管理全部”入口联动打开。
2. 默认模板清单：初始迁移的 2-3 条具体模板内容范围（需求澄清/周报/复盘等）。
3. 增强规划优先级：搜索、分类/标签、导入导出能力的取舍与顺序。
4. 数据导出格式：如支持导入导出，是否采用 JSON/Markdown 两种形式。
